# FTF Contribution Types

This document explains the different types of contributions accepted for Free Tools Friday.

---

## Tools

**What they are:** PowerShell modules, scripts, small utilities, and automation tools.

**Where they live:** Tools with real code live in their own repositories under the Peach-Security GitHub organization.

**How to contribute:** Submit PRs to the tool's own repository.

**Examples:**
- PowerShell discovery modules
- Security scanning scripts
- Automation utilities

---

## Resources

**What they are:** Templates, checklists, and client-ready materials.

**Where they live:** This repository under `/releases/resources/`

**How to contribute:** Submit PRs to this repository following the file structure conventions.

**Examples:**
- Policy templates (AUP, security policies)
- Assessment checklists
- Client communication templates

---

## Guides

**What they are:** Practical documentation, playbooks, and how-to resources.

**Where they live:** This repository under `/releases/guides/`

**How to contribute:** Submit PRs to this repository following the file structure conventions.

**Examples:**
- Hardening guides
- Implementation playbooks
- Best practice documentation

---

## Ideas

**What they are:** Suggestions for future FTF releases.

**How to contribute:** Submit and vote on ideas at the FTF ideas board: https://www.peachsecurity.io/ftf

Your votes help prioritize what gets built next.
